Codex に Mac の操作を許可する方法

作成日: 2026-06-17

カテゴリ: ai-coding-agents

元の質問

codexにMacの操作を許可する方法は？

回答

Codex に Mac の画面を見せたり、クリック・入力・アプリ操作をさせたりするには、Codex app の Computer Use を使います。通常のファイル編集やシェル実行の権限とは別で、macOS 側の権限と Codex 側のアプリ承認を両方設定します。

結論だけ書くと、Codex の Settings で Computer Use をインストールし、macOS の「画面収録」と「アクセシビリティ」を Codex に許可し、実行時に対象アプリの使用を承認する、という流れです。

設定手順

  1. Codex app を開きます。
  2. Settings を開きます。通常はアプリメニュー、または Command + , から開けます。
  3. Computer Use を開き、未インストールなら Install します。
  4. macOS の権限ダイアログが出たら、Screen Recording（画面収録） と Accessibility（アクセシビリティ） を許可します。
  5. ダイアログが出ない、または後で確認する場合は、macOS の システム設定 > プライバシーとセキュリティ を開きます。
  6. 画面収録 で Codex をオンにします。これは Codex が対象アプリの画面を見るために必要です。
  7. アクセシビリティ で Codex をオンにします。これは Codex がクリック、入力、メニュー操作を行うために必要です。
  8. macOS に再起動やアプリの再起動を求められた場合は、Codex app を終了して開き直します。

Codex に操作を依頼する書き方

設定後は、プロンプトで Computer Use を使うことと、操作対象を明示します。OpenAI の Codex ドキュメントでは、@Computer や @AppName を指定する方法が案内されています。

プロンプト例（言語名: text）:

@Computer を使って Safari を開き、このローカルページの表示崩れを確認してください。
対象は http://localhost:3000/ だけです。
フォーム送信やアカウント設定変更はしないでください。
確認後、見つけた問題とスクリーンショット上の位置を報告してください。

Chrome を使うなら、次のように対象アプリを狭く指定します。

プロンプト例（言語名: text）:

@Chrome を使って、ログイン後の設定画面を開き、表示だけ確認してください。
設定変更、削除、送信、購入、認証情報の表示はしないでください。
確認が必要な操作が出たら、実行前に止まってください。

macOS 権限と Codex の承認は別

macOS の「画面収録」と「アクセシビリティ」を許可しても、Codex がすべてのアプリを勝手に操作できる状態になるわけではありません。Codex 側でも、タスク中に対象アプリを使う前に承認を求めます。

よく使うアプリなら Always allow を選べます。ただし、自動で使わせてもよいアプリだけに限定するのが安全です。後から取り消したい場合は、Codex の Settings > Computer Use で許可済みアプリを管理し、macOS 側の権限は システム設定 > プライバシーとセキュリティ から取り消します。

サンドボックス権限とも別

Computer Use は「画面を見る」「アプリを操作する」ための権限です。一方で、ファイル編集、シェルコマンド、ネットワークアクセスは Codex の sandbox と approval policy の管理下にあります。

たとえば Codex が Finder やブラウザを操作できても、リポジトリ外のファイル編集やネットワークを使うコマンドは、別途サンドボックスや承認設定に従います。Mac 操作を許可することと、danger-full-access のような広い実行権限を与えることは同じではありません。

ロック中の Mac で使いたい場合

macOS では、Codex app の Settings > Computer Use から locked computer use を有効にできる場合があります。これは、Mac がロックされた後でも、信頼済みの実行中 Computer Use タスクに限って Codex が一時的に操作を継続できるようにする機能です。

汎用的なリモート解除機能ではありません。使う場合も、対象タスクを明確にし、アカウント、支払い、認証情報、セキュリティ設定のような操作は避けるのがよいです。

できないこと・注意点

  • Codex は macOS の管理者認証や、セキュリティとプライバシーの許可ダイアログを代わりに承認できません。そこは人間が操作します。
  • Computer Use はターミナルアプリや Codex 自体の自動操作には使えません。セキュリティ境界を迂回する可能性があるためです。
  • ブラウザを操作させる場合、ログイン済みサイトでは自分のアカウントで操作したのと同じ扱いになります。送信、購入、削除、設定変更は明示的に止めるか、実行前承認にします。
  • パスワード、二要素認証コード、秘密鍵、個人情報が表示されるアプリは、必要なときだけ開き、Codex に見せる範囲を絞ります。
  • Web アプリのローカル確認だけなら、まず Codex の in-app browser を使う方が狭い権限で済む場合があります。
