このプロジェクトの使い方

作成日: 2026-06-12

カテゴリ: project

元の質問

このプロジェクトの使い方は？

回答

このプロジェクトは、技術的な疑問を 1 テーマ 1 ページの静的 HTML 記事として残すためのノートです。普段の使い方は、質問を Codex や Claude Code に渡し、エージェントが記事ファイルとトップページの記事カードを更新してコミットし、人間が差分を確認してから push・公開する流れです。

基本の流れ

  1. 技術的な疑問を Codex / Claude Code に渡します。
  2. エージェントが必要に応じて一次情報を確認し、notes/<category>/<slug>.html に記事を作ります。
  3. エージェントが index.html の記事一覧にカードを追加します。
  4. エージェントが作業内容をコミットします。
  5. 人間がコミットと差分を確認します。
  6. ローカル運用では、人間が確認後に push します。
  7. Codex cloud などの遠隔運用では、作業ブランチと PR を作り、人間が PR を確認してからマージします。

質問だけ送れば記事化される

このリポジトリでは、ユーザーが技術的な疑問だけを送った場合、特に断りがなければ記事作成依頼として扱います。毎回「記事にして」と書く必要はありません。このプロジェクト自身の使い方や運用ルールに関する質問も記事化対象です。

会話だけで済ませたい場合は、「説明だけ」「記事にしないで」「相談だけ」のように明示します。

質問例（言語名: text）:

uv.lock はいつ更新される？
GitHub Actions の permissions は何を指定すべき？
Python の pathlib.Path と os.path はどう使い分ける？
このプロジェクトの使い方は？

記事の置き場所と書式

記事は notes/_template/article.html を元に作り、カテゴリ名と記事ファイル名は英小文字のケバブケースにします。記事ページには、タイトル、作成日、カテゴリ、元の質問、構造化された本文、参照 URL、トップページへの戻りリンクを置きます。

記事固有の CSS は原則使わず、記事ページからは ../../styles/main.css を読み込みます。ビルドツールや静的サイトジェネレータは使いません。

ローカルで読む方法

ビルド工程はありません。ローカルでは `index.html` をブラウザで開けば記事一覧を確認できます。記事ページはトップページのカードから移動します。

公開の仕組み

main ブランチにマージされると GitHub Pages のワークフローが動きます。ワークフローは公開対象だけを _site に集め、index.html、styles/、images/、notes/ をデプロイします。ただし notes/_template/ は除外されます。

AGENTS.md、CLAUDE.md、README.md、plan.md、cleanup.sh などの運用ドキュメントは公開対象ではありません。

注意点

  • 記事は日本語で書きます。
  • 推測を書く場合は「推測」と明示します。
  • 機密情報、個人情報、未公開の認証情報は記事に含めません。
  • main ブランチへ直接プッシュせず、人間の確認を通して公開します。
  • ファイル削除が必要な場合は直接削除せず、cleanup.sh に削除コマンドを追記して確認を待ちます。
