EZproxyとは？

作成日: 2026-07-01

更新日: 2026-07-01

カテゴリ: library-tech

元の質問

EZproxyとは？

回答

EZproxy は、図書館や大学などが契約している電子ジャーナル、文献データベース、電子ブックを、学外や館外から使えるようにするためのアクセス仲介システムです。OCLC が提供している製品で、利用者とコンテンツ提供元の間に入り、利用者が機関の正規ユーザーであることを確認したうえで、コンテンツ提供元には機関が登録した EZproxy の IP アドレスからのアクセスとして見せます。

短く言えば、「出版社やデータベース側が IP アドレスで契約機関を判定している世界」と、「利用者が自宅や出張先からアクセスする現実」をつなぐ仕組みです。

なぜ必要になるのか

多くの学術データベースや電子ジャーナルは、契約機関のネットワークから来たアクセスを許可する形で運用されています。図書館内や大学内の PC からなら、出版社側は接続元 IP アドレスを見て「この機関の契約範囲内」と判断できます。

しかし、利用者が自宅の回線やモバイル回線から直接アクセスすると、出版社側から見えるのは利用者個人の IP アドレスです。その IP アドレスは契約機関の登録済み IP ではないため、フルテキストや有料データベースへのアクセスが拒否されます。

EZproxy はこの問題を、機関側で利用者認証を行い、認証済みのリクエストだけを EZproxy 経由で出版社側へ送ることで解決します。

基本的な流れ

  1. 利用者が図書館サイト、ディスカバリーサービス、または EZproxy 用のリンクから電子リソースへアクセスします。
  2. EZproxy が利用者を機関の認証方式で確認します。OCLC の説明では、SAML や LDAP などの認証方式に対応します。
  3. 認証後、EZproxy は利用者の代わりにコンテンツ提供元へリクエストを送ります。
  4. コンテンツ提供元は、接続元を利用者個人の IP ではなく、機関が登録した EZproxy の IP として見ます。
  5. 提供元から返ってきたページや PDF などを、EZproxy が利用者のブラウザへ返します。

このため、利用者から見ると「大学・図書館のログインを通すと、学外からでも契約リソースが読める」ように見えます。

東京大学の場合

東京大学の EZproxy ホストは utokyo.idm.oclc.org です。東京大学附属図書館のデータベース一覧では、E-journal & E-book Portal、TREE、JapanKnowledge、Web of Science、Scopus などの項目に 利用する(EZproxy) のリンクがあり、リンク先は https://utokyo.idm.oclc.org/login?url=... という形になっています。

したがって、東大所属者が学外から使う場合は、基本的には次のどちらかから入ります。

  • 東京大学附属図書館のデータベース一覧で、目的のデータベースの 利用する(EZproxy) をクリックする。
  • 学外からの利用の案内に従い、E-journal & E-book Portal や TREE から EZproxy 経由で入る。

URL の形だけを書くと、汎用的には次の形式です。

https://utokyo.idm.oclc.org/login?url=アクセスしたい電子リソースのURL

たとえば、東京大学附属図書館のデータベース一覧にある E-journal & E-book Portal の EZproxy リンクは、次のような形式です。

https://utokyo.idm.oclc.org/login?url=https%3A%2F%2Fvs2ga4mq9g.search.serialssolutions.com%2F

ただし、任意の URL を付ければ何でも使えるわけではありません。東京大学が契約し、EZproxy の設定対象になっている電子リソースについて、東京大学アカウント等で認証された所属者が利用するための入口だと考えるのが正確です。

URL 書き換えが中心になる

EZproxy の特徴は、利用者のブラウザ全体をプロキシ設定に変えさせるのではなく、電子リソースの URL を EZproxy 経由の URL に書き換えていく点です。OCLC の文書では、この方式を URL rewriting として説明しています。

たとえば、実際のデータベースが次の URL だとします。

https://www.somedb.com/index.html

Proxy by hostname 方式では、EZproxy 配下のホスト名へ次のような形で対応づけられます。

https://www-somedb-com.ezproxy.example.edu/index.html

こうすると、ページ内のリンクも EZproxy 経由に書き換えられ、利用者はブラウザの通常操作のまま契約リソース内をたどれます。従来型の汎用プロキシのように、ブラウザ側でプロキシサーバーを手動設定する必要がない点が運用上大きいです。

データベーススタンザとは何か

EZproxy では、どの電子リソースをプロキシ対象にするかを設定で定義します。このリソースごとの設定単位が、database stanza と呼ばれるものです。スタンザには、対象サービスのホスト名、URL、ドメイン、必要な追加ディレクティブなどが書かれます。

重要なのは、電子リソースは単一のトップページだけで完結しないことです。検索画面、本文画面、画像配信、PDF 配信、認証用サブドメインなど、複数のホストや URL が関係します。スタンザが不足していると、トップページは開けても本文や PDF で認証が切れる、画像だけ出ない、別ドメインへ移動した瞬間に学外アクセス扱いになる、といった問題が起こります。

VPN や SSO との違い

VPN は、利用者の端末を機関ネットワークに入ったように扱う広いネットワーク接続です。電子リソース以外の通信にも影響するため、ネットワーク設計や端末管理の対象が広くなります。

SSO は、利用者が誰かを確認する認証基盤です。SAML や LDAP などでログインをまとめられますが、出版社側が「契約機関からの IP アクセス」を前提にしている場合、SSO だけでは出版社に機関 IP として見せる役割は果たしません。

EZproxy は、SSO などで利用者を確認したあと、電子リソースへの Web アクセスだけを機関の認可済み IP 経由にする道具です。したがって、VPN の代替というより、図書館の電子リソース提供に特化したアクセス仲介層と考えるほうが正確です。

運用上の注意点

  • 契約条件に応じて、学部、身分、所属、利用者グループごとにアクセス範囲を分ける必要があります。
  • 不正利用や漏えいした認証情報を検知するため、ログ、利用量、同時接続、異常なダウンロードなどの監視が重要です。
  • 出版社側のサイト構成変更に合わせて、スタンザの更新が必要になることがあります。
  • 利用ログは電子リソース評価に役立ちますが、誰が何を読んだかに近い情報を含み得るため、プライバシーと保存期間の設計が必要です。
  • EZproxy は匿名化サービスではありません。目的は、契約機関の正規利用者に契約リソースを届けることです。

まとめ

EZproxy は、図書館契約の電子リソースをリモート利用させるための「認証 + IP ベースアクセス仲介 + URL 書き換え」の仕組みです。利用者には学外からでもアクセスしやすい導線を提供し、出版社側には契約機関の認可済み IP からのアクセスとして見せます。図書館システムの文脈では、単なる Web プロキシではなく、電子リソース契約、認証基盤、URL 構造、利用ログ、プライバシー管理が交差する場所にあるシステムです。
