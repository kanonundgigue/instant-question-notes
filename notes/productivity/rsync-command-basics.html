rsync コマンドの使い方は？

作成日: 2026-07-10

カテゴリ: productivity

元の質問

rsyncコマンドの使い方は？

回答

rsync は、ローカルまたはリモートのファイルを効率よくコピー、同期するコマンドです。公式 manpage では、高速で多用途なファイルコピー用ツールであり、コピー元とコピー先の差分だけを送る delta-transfer algorithm が特徴だと説明されています。[1]

まず覚える基本形は、次の 2 つです。

コード例（言語名: shell）:

# コピー元ディレクトリの「中身」をコピー先へ同期する
rsync -av source/ dest/

# 実行前に、何が起きるかだけ確認する
rsync -avn source/ dest/

-a は archive mode、-v は詳細表示、-n は dry run です。archive mode は再帰コピーに加えて、シンボリックリンク、権限、時刻、所有者、グループなどを保つための複数オプションをまとめたものです。[1] 実務では、まず rsync -avn ... で確認し、問題なければ n を外して実行する癖を付けると安全です。

末尾のスラッシュが一番重要

rsync で最も間違えやすいのは、コピー元の末尾に / を付けるかどうかです。公式 manpage は、コピー元末尾の / は「そのディレクトリの中身をコピーする」という意味に近く、付けない場合は「ディレクトリ名ごとコピーする」と考えられる、と説明しています。[1]

コマンド 結果 使う場面
rsync -av source/ dest/ source の中身が dest/ に入る。 フォルダの中身だけを同期したい。
rsync -av source dest/ dest/source/ が作られる。 フォルダ名ごとコピーしたい。

迷ったら、「コピー先に source というディレクトリ名を作りたいか」を考えます。作りたくないなら、コピー元を source/ にします。

ローカルでフォルダを同期する

同じ PC 内で、作業ディレクトリを外付けディスクや別フォルダへコピーする場合は、両方とも普通のパスで指定します。

コード例（言語名: shell）:

# まず確認する
rsync -avn ~/work/project/ /Volumes/Backup/project/

# 表示内容に問題がなければ実行する
rsync -av ~/work/project/ /Volumes/Backup/project/

コピー先が存在しない場合は作成されます。ただし、親ディレクトリが存在しない場合の挙動は環境やオプションに依存するため、最初はコピー先の親ディレクトリを先に作っておくのが分かりやすいです。

SSH 経由で別マシンへ送る

別マシンへ送る場合は、コピー先を user@host:/path/ の形で書きます。公式 manpage は、リモート shell 経由の push と pull の構文を示しており、現代的な rsync は通常 SSH を通信に使うと説明しています。[1]

コード例（言語名: shell）:

# 手元からサーバーへ送る
rsync -av ~/work/project/ user@example.com:/home/user/project/

# サーバーから手元へ持ってくる
rsync -av user@example.com:/home/user/project/ ~/work/project/

SSH のポートを指定したい場合は、-e で SSH コマンドを渡します。

コード例（言語名: shell）:

rsync -av -e 'ssh -p 2222' ./data/ user@example.com:/data/project/

ミラーリングには --delete を使う

普通の rsync -av は、コピー元から消えたファイルをコピー先から自動では消しません。コピー先をコピー元と同じ状態にそろえたい場合は、--delete を付けます。公式 manpage でも --delete は「コピー先ディレクトリの余分なファイルを削除する」オプションとして説明されています。[1]

コード例（言語名: shell）:

# 必ず dry run で削除予定を確認する
rsync -av --delete --dry-run source/ dest/

# 問題なければ実行する
rsync -av --delete source/ dest/

--delete は便利ですが、コピー元やコピー先を間違えると大きく消します。初回、外付けディスク、共有サーバー、本番データでは、必ず --dry-run と組み合わせて削除対象を確認します。

よく使うオプション

オプション 意味 使いどころ
-a archive mode。 ディレクトリを普通に同期する基本。
-v 詳細表示。 何が転送されたか見たい。
-n / --dry-run 変更せずに試算する。 初回実行、--delete 使用時。
-h サイズを読みやすく表示する。 人間がログを見る。
--progress 転送中の進捗を表示する。 大きなファイルのコピー。
--exclude='pattern' 指定パターンを除外する。 cache、生成物、巨大ファイルを送らない。
-z 転送データを圧縮する。 低速なネットワーク越しの転送。高速 LAN や圧縮済みファイルでは不要なこともある。

除外パターンを書く

--exclude は、転送したくないファイルやディレクトリを指定します。公式 manpage では、--exclude=PATTERN は filter rule の簡易形で、複数指定したい場合はオプションを繰り返すか、--exclude-from=FILE を使うと説明されています。[1]

コード例（言語名: shell）:

rsync -av \
  --exclude='.git/' \
  --exclude='node_modules/' \
  --exclude='outputs/' \
  ./project/ /Volumes/Backup/project/

除外ルールは後から見直しやすいように、バックアップ用スクリプトや設定ファイルにまとめると事故を減らせます。

実務での安全な手順

  1. コピー元とコピー先を絶対パスで書く。
  2. コピー元末尾の / の意味を確認する。
  3. まず -n または --dry-run を付けて実行する。
  4. 表示された転送、削除、除外の内容を読む。
  5. 問題がなければ -n だけ外して本実行する。

最初に使う定型は、次で十分です。

コード例（言語名: shell）:

# 中身を同期。削除はしない
rsync -avh --dry-run source/ dest/
rsync -avh source/ dest/

# 完全なミラーにする。削除予定を必ず確認する
rsync -avh --delete --dry-run source/ dest/
rsync -avh --delete source/ dest/
