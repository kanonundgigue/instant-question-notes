複数メールアドレスと AI を使うメーラーの選び方

作成日: 2026-06-16

更新日: 2026-06-17

カテゴリ: productivity

元の質問

複数のメールアドレスを簡単に切り替えられるメーラーで、無料でAIが使える（すでに契約済みのOpenAIやAnthropicと接続できる）のは？ APIの課金が必要なんだったら、sparkのAIに課金するのでも変わらない？ AIが有料のサービスも視野に比較をしなおして。

回答

2026-06-16 時点では、AI に課金してよいなら Spark +AI も十分に有力です。さらに、Apple デバイスを使っていて Apple Mail で足りるなら、Apple Mail + Apple Intelligence も英語メール添削の強い候補になります。最初の条件を「無料で AI も使いたい」と厳密に読むなら Thunderbird + ThunderAI が第一候補ですが、OpenAI API や Claude API の従量課金が別途必要になるなら、Spark の有料プランに払うのと費用面の意味はかなり近くなります。

違いは「どこに払うか」と「どの環境に寄せるか」です。Spark +AI や Canary Mail は、メーラー側の月額・年額に AI 利用枠が含まれるので設定が楽です。Apple Intelligence は対応 Apple デバイスと対応 OS があれば、Mail の中で Writing Tools、Smart Reply、要約、Priority Messages を使えます。Thunderbird + ThunderAI は、メーラー本体は無料で、AI だけ OpenAI API / Claude API / Gemini / Ollama などを自分で選びます。

結論

  • いちばん普通におすすめしやすい: Spark。無料プランでも複数アカウント管理はでき、Plus 以上で Spark +AI と AI Assistant が使えます。AI 課金込みでメール体験全体を買う選択です。
  • Apple デバイスだけで英語添削したい: Apple Mail + Apple Intelligence。対応 Mac / iPhone / iPad を持っているなら、追加のメーラー課金なしで Writing Tools と Mail の AI 機能を試す価値があります。
  • 安い内蔵 AI を優先: Canary Mail。Free でも無制限メールアカウント、Growth は年額 36 ドル相当で AI Copilot が入ります。年額課金または買い切り前提です。
  • OpenAI / Claude / ローカル LLM を自分で選びたい: Thunderbird + ThunderAI。月額メーラーを増やさず、API キーや Ollama を使って自分で構成します。
  • AI 検索・ワークフローまで重視: Shortwave。強力ですが、個人の通常メーラーとしては価格帯が上がります。

候補比較

候補 合う点 注意点
Spark Free でも無制限メールアカウント、統合受信箱、Smart Inbox が使える。Plus は月額 10 ドル、年額なら月あたり 8.25 ドル相当で Spark +AI と AI Assistant が入る。 AI は Spark 側の利用枠と仕様に乗る。OpenAI / Anthropic の API キーを自分で選んで使う BYOK 型ではない。
Canary Mail Free でも統合受信箱と無制限メールアカウント。Growth は年額 36 ドル相当で AI Copilot、作成・返信・要約・優先度付けが入る。 月額払いではなく年額または買い切り。AI のモデル選択や API 持ち込みより、Canary の内蔵 AI を使う発想。
Apple Mail + Apple Intelligence 対応 Apple デバイスなら Mail で Writing Tools、Smart Reply、要約、Priority Messages を使える。英語添削、丁寧化、短文化、返信草案に向く。 Apple デバイス、対応 OS、対応言語・地域が前提。OpenAI / Anthropic API キーを持ち込む BYOK 型ではなく、Apple Intelligence と任意の ChatGPT 連携を使う。
Thunderbird + ThunderAI Thunderbird 本体は無料。複数アカウントを個別表示または統合受信箱で扱える。ThunderAI は ChatGPT Web、OpenAI API、Claude API、Gemini、Ollama などを明記している。 標準機能ではなくアドオン構成。API 利用料、API キー、送信データ、モデル選択を自分で管理する。
eM Client メール、カレンダー、連絡先、タスクをまとめられる。AI は返信作成、本文生成、要約、言い換えなどに使える。 無料版は個人・非商用利用でメールアカウント 2 個まで。AI の現在の provider は ChatGPT と説明されており、Claude 持ち込み用途ではない。
Shortwave AI 検索、AI フィルタ、添付分析、AI 返信、AI による受信箱整理などが強い。Claude 系モデルの intelligence level を明示している。 Business は年額払いで月あたり 24 ドルから。普通の複数アカウントメーラーというより、AI ワークフロー込みの高機能メール環境。

Spark に払うのと API に払うのは同じか

費用面だけなら、かなり近いです。OpenAI の API は ChatGPT Plus / Business / Enterprise / Edu とは別課金で、Anthropic も Claude API のモデル別料金を公開しています。つまり「ChatGPT や Claude はすでに契約しているから、メール AI は無料で使える」とは限りません。外部メーラーから API 接続する場合は、別途 API 利用料が発生する前提で見ます。

そのため、日常的にメールを AI で要約・返信・整形するなら、Spark Plus のような固定月額のほうが分かりやすい場面があります。Spark は無料プランで複数アカウント管理を使い、AI が必要になったら Plus に上げる、という移行が自然です。Spark +AI はデフォルトでは無効で、設定から管理できるとも説明されています。

逆に、月に数十回だけ短い返信案を作る程度なら、Thunderbird + ThunderAI + 安い API モデルのほうが低コストになり得ます。さらに、Claude を使いたい、Ollama でローカルモデルを使いたい、メール本文をどの provider に送るか自分で決めたい、という要求があるなら BYOK 型の価値が出ます。

用途別の選び方

  • 複数アドレスを気持ちよく扱い、AI も自然に使いたい: Spark Plus。固定費を許容できるなら、最も無難です。
  • Mac / iPhone / iPad の標準 Mail で足りる: Apple Mail + Apple Intelligence。英語添削と返信補助が中心なら、まず追加課金なしで試す価値があります。
  • できるだけ安く内蔵 AI を使いたい: Canary Mail Growth。年額 36 ドル相当で AI Copilot が入るため、価格だけならかなり強いです。
  • Gmail / Outlook のメールを AI で検索・整理・自動化したい: Shortwave。価格は上がりますが、AI 検索やフィルタが目的なら候補になります。
  • 既存の OpenAI / Anthropic API、Gemini、Ollama を自分で使いたい: Thunderbird + ThunderAI。最も自由ですが、設定と運用の手間も最も大きいです。
  • メールアカウントが 2 個までで、昔ながらのデスクトップ統合環境がよい: eM Client。AI もありますが、複数アカウントが多い人は無料版の 2 アカウント制限に当たりやすいです。

英語メールの添削が主目的なら

英語メールの添削、言い換え、トーン調整が主目的なら、Apple デバイスを使っている人は Apple Mail + Apple Intelligence を先に試します。Apple の Writing Tools は、選択した文章の校正、Rewrite、Friendly / Professional / Concise へのトーン変更、要約に対応し、Mail では Smart Reply やスレッド要約も使えます。メール作成画面の中で「自然な英文に直す」「丁寧にする」「短くする」を行えるので、英語添削用途に合います。

Apple Mail 以外のメーラーを使いたい、Windows / Android も含めたい、または複数アカウントの処理体験をアプリ側で買いたいなら、次の候補は Spark Plus か Canary Mail Growth です。費用を抑えたいなら Canary Mail Growth が有力です。年額 36 ドル相当で AI Copilot が入り、作成・返信・要約系の用途に合います。月額で始めたい、複数アカウント管理も含めて全体の使いやすさを優先したいなら Spark Plus が無難です。

一方、添削の品質や文体を細かく制御したいなら Thunderbird + ThunderAI も残ります。OpenAI API、Claude API、Gemini、Ollama などを選べるため、「短く自然な英語」「査読者への丁寧な返信」「事務連絡として硬すぎない英語」などのカスタムプロンプトを自分で作れます。ただし、API キー設定とモデル課金の管理が必要なので、単に英語メールを直したいだけなら少し手間が重いです。

Apple Intelligence の ChatGPT 連携は、ChatGPT アカウントなしでも使え、既存の無料または有料アカウントを接続することもできます。ただし、これは Apple Intelligence の拡張として ChatGPT を使う仕組みであり、OpenAI API キーや Anthropic API キーをメーラーに直接持ち込む仕組みではありません。すでに ChatGPT や Claude の画面利用サブスクリプションを持っているなら、少量の添削は本文をコピーして直接直すだけでも足ります。

Spark などを使いながら Apple Intelligence で添削できるか

原理的には可能です。Apple の説明では、Apple Intelligence の Writing Tools は、Apple 純正アプリだけでなく、サードパーティアプリや Web サイトを含む「書く場所の多く」で利用できます。つまり、対応 Mac で Spark のメール作成欄に英文を書き、本文を選択して Control-click し、Show Writing Tools から Proofread、Rewrite、Professional、Concise などを使える可能性があります。

ただし、これは Spark の AI 機能そのものではなく、macOS 側の Writing Tools をテキスト入力欄にかける使い方です。Spark の作成欄が標準的なテキスト編集コンポーネントとして扱われれば使いやすい一方、アプリ側が独自の入力欄を使っていて Writing Tools メニューが出ない場合は、その場では使えません。その場合は、本文を Notes、TextEdit、Apple Mail、または ChatGPT などに一度移して添削し、Spark に戻す運用になります。

この併用がうまく動くなら、Spark の無料または Plus のメール管理機能を使いながら、英文添削だけ Apple Intelligence に任せる構成が取れます。Spark +AI に課金するかどうかは、Spark 内蔵の返信草案、要約、スマート処理まで必要かで判断します。単純な英語添削だけなら、まず Apple Intelligence の Writing Tools を試してからで十分です。

プライバシーと運用の見方

AI メーラーでは、要約・返信生成・検索のためにメール本文や添付情報が AI 処理に渡る可能性があります。Spark や Canary のような内蔵 AI は設定が簡単な一方で、どのモデル・経路で処理されるかはサービス側の仕様に従います。Apple Intelligence は対応デバイス・対応 OS 内の機能として使え、ChatGPT 連携を使う場合は、ChatGPT に送る内容をユーザーが選ぶ設計です。Thunderbird + ThunderAI は provider を選べる一方で、API キー管理やプロンプト送信範囲の責任が自分に寄ります。

会社・学校・顧客とのメールを扱うなら、送ってよい内容、送ってはいけない内容、AI provider のデータ保持、組織の規程を先に確認します。個人用途でも、全メールを自動で処理するのではなく、要約・校正・返信草案など明確な操作時だけ AI に渡す設定から始めるのが現実的です。

実務上の判断

質問の条件なら、比較し直した結論は Apple Mail で足りるなら Apple Intelligence、メーラー体験ごと買うなら Spark です。Apple デバイス中心で英語添削が主目的なら Apple Intelligence を先に試します。Apple Mail ではなく、複数アカウントの整理、通知、スマート受信箱まで含めて作業環境を整えたいなら Spark を第一候補に上げ直してよいです。

コストを強く見るなら Canary Mail Growth、AI 検索や自動整理まで見るなら Shortwave、モデル選択とローカル LLM まで含めて自分で組みたいなら Thunderbird + ThunderAI、という分岐で考えます。
