Mac (Apple Silicon) で apt-get したい場合は？

作成日: 2026-07-04

カテゴリ: productivity

元の質問

Mac (apple silicon)でapt-getしたい場合は？コマンドインストールできる？

回答

macOS で apt-get をそのまま使う、という発想はいったん分けたほうがよいです。apt-get は Debian/Ubuntu 系 Linux の APT パッケージを扱うコマンドラインツールです。Debian の man page でも、apt-get は APT library を使うパッケージ操作用コマンドとして説明されています。[1] Mac ネイティブのコマンドを入れたいなら、通常は Homebrew か MacPorts を使います。Linux 環境そのものが必要なら、Docker などの Linux コンテナ、または仮想マシンの中で apt-get を使います。

結論

  • Mac に wget、ffmpeg、git、gfortran などのコマンドを入れたいだけなら、まず brew install パッケージ名 を使う。
  • MacPorts 前提のソフト、variant 指定、Homebrew と別 prefix の依存関係ツリーが必要なら、sudo port install パッケージ名 を使う。
  • README に sudo apt-get install ... と書かれていて、その Linux 手順をそのまま再現したいなら、Ubuntu/Debian コンテナか VM の中で実行する。

Homebrew で入れる

Apple Silicon Mac で最も手軽な選択肢は Homebrew です。Homebrew 公式のインストールページでは、Apple Silicon の標準 prefix は /opt/homebrew、Intel Mac は /usr/local と説明されています。公式トップページにも、Homebrew は macOS、Linux、WSL をサポートし、コマンドラインツールやアプリケーションを入れるものとして案内されています。[2][3]

コマンド例（言語名: shell）:

# Homebrew を入れる
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# インストール後、表示された指示に従って shell に brew を認識させる
eval "$(/opt/homebrew/bin/brew shellenv)"

# apt-get install 相当
brew install wget
brew install ffmpeg

# GUI アプリなら cask
brew install --cask firefox

# 更新
brew update
brew upgrade

Homebrew は通常の brew install で sudo を使いません。Homebrew の FAQ では、パッケージ定義と Homebrew 本体の更新に brew update、インストール済みパッケージの更新に brew upgrade を使う手順が示されています。[4]

MacPorts で入れる

MacPorts も macOS 用のパッケージ管理です。MacPorts の pkg インストーラは MacPorts 本体の導入、shell 環境の設定、ports tree と MacPorts base の selfupdate まで行い、動作確認として新しいターミナルで port version を試す流れが説明されています。[5] インストール操作は sudo port install ... が基本で、MacPorts Guide では sudo port install apache2 -preforkmpm +workermpm のような例が示されています。[6]

コマンド例（言語名: shell）:

# MacPorts の更新
sudo port selfupdate

# パッケージ検索
port search wget
port info wget

# apt-get install 相当
sudo port install wget

# variant を見てから入れる
port variants apache2
sudo port install apache2 +openldap

apt-get そのものが必要な場合

apt-get で入るものは、基本的に Debian/Ubuntu などの Linux 環境に対して入ります。Mac の /usr/bin や /opt/homebrew に Debian パッケージを直接展開して macOS ネイティブコマンドとして使う、という運用ではありません。Linux の手順を再現したいなら、Docker Desktop などで Ubuntu/Debian コンテナを動かし、その中で apt-get を実行します。Docker 公式は Mac 用 Docker Desktop の Apple Silicon 版を提供しており、Apple Silicon Mac と Intel Mac それぞれのダウンロード導線とシステム要件を示しています。[7]

コマンド例（言語名: shell）:

# Docker Desktop を起動してから、Ubuntu コンテナに入る
docker run --rm -it ubuntu:24.04 bash

# ここから先はコンテナ内の Linux
apt-get update
apt-get install -y wget build-essential
wget --version

この方法で入れた wget や build-essential はコンテナ内のものです。Mac の通常ターミナルから直接使うコマンドを入れたい場合は、同名または近いパッケージを Homebrew や MacPorts で探します。

apt-get と Mac の対応表

やりたいこと Linux の感覚 Mac での置き換え
パッケージ一覧を更新する sudo apt-get update brew update または sudo port selfupdate
コマンドを入れる sudo apt-get install wget brew install wget または sudo port install wget
全体を更新する sudo apt-get upgrade brew upgrade または sudo port upgrade outdated
Debian/Ubuntu の手順をそのまま動かす apt-get 前提の README Ubuntu/Debian コンテナまたは VM 内で実行

Apple Silicon で気をつけること

  • Homebrew は Apple Silicon では /opt/homebrew が標準なので、古い Intel Mac 向け記事にある /usr/local/bin/brew 前提の記述をそのまま使わない。
  • Intel 専用バイナリしかないソフトは Rosetta 2 や --platform linux/amd64 が必要になる場合がある。ただし、まずは arm64 ネイティブの formula、port、コンテナイメージを探す。
  • Homebrew と MacPorts を併用する場合は、PATH の順番で同名コマンドが変わる。普段使いの系統を 1 つ決め、別系統は必要な作業だけで使う。

まとめ

「Mac で apt-get したい」が「コマンドをインストールしたい」という意味なら、Apple Silicon Mac ではまず Homebrew の brew install を使うのが最短です。MacPorts が向くソフトなら sudo port install も選択肢です。apt-get というコマンド自体が必要な場合だけ、Ubuntu/Debian コンテナや VM の中で使う、と切り分けるのが混乱しにくいです。
