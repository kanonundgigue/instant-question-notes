GrADS の XQuartz 画面が黒いままのときの対処法

作成日: 2026-07-04

カテゴリ: climate-data

元の質問

GrADS の XQuartz 画面表示問題はどうしたら解決できる？バージョンの噛み合わせの問題？

回答

MacPorts 版 GrADS で draw map や d u を実行しても XQuartz のウィンドウが黒いままなのに、printim で保存した PNG は正しく描ける場合、まず疑うべきなのはデータや GrADS コマンドではなく、XQuartz のライブ表示経路です。MacPorts の grads port は GrADS 2.2.1 を提供し、地球科学データの対話的な可視化ツールとして説明されています。[1]

この症状では、最短の対処として GrADS の表示 backend を既定の Cairo から X11 に切り替えます。MacPorts 版の手元の環境では /opt/local/share/grads/udpt に Cairo と X11 の表示 plugin があり、grads -d X11 で X11 backend を明示できます。

export GADDIR=/opt/local/share/grads
grads -d X11 -lp

毎回打たなくてよいようにする

ライブ表示を使うたびに -d X11 を付け忘れないよう、~/.zshrc に専用関数を置きます。通常の grads コマンドは残し、XQuartz の画面表示を使うときだけ gradsx を使うのが安全です。

export GADDIR="/opt/local/share/grads"

gradsx() {
    command grads -d X11 "$@"
}

設定後、新しいターミナルを開くか、次のように読み込み直します。

source ~/.zshrc
gradsx -lp

動作確認の順番

まず XQuartz を起動し、通常のターミナルから gradsx -lp を実行します。GrADS のプロンプトが出たら、簡単な描画を試します。

# GrADS プロンプト内
clear
set dbuff off
draw rec 1 1 10 7
draw string 3 4 GrADS X11 test

データファイルを開ける状態なら、実データでも確認します。

# GrADS プロンプト内
open /tmp/sample.ctl
set gxout contour
d u
draw map

画面が表示されるなら、原因の中心は GrADS のデータ処理ではなく、Cairo 表示 backend と XQuartz/macOS の組み合わせにあります。推測を含む言い方をすれば、XQuartz 画面へのライブ描画更新だけが詰まっており、GrADS 側の描画内容は作れている、という状態です。

なぜバージョンの噛み合わせを疑うのか

XQuartz 2.8.5 は 2023-01-26 にリリースされており、変更点には resize handling や window content rendering に関する修正が含まれています。[2] 一方、2026-06-19 には XQuartz 2.8.6_rc2 が公開され、基盤プロジェクトとして cairo 1.18.4 などが列挙されています。[3] そのため、macOS の新しい環境で XQuartz 2.8.5 と GrADS の Cairo 表示がうまく噛み合っていない、という見立ては十分ありえます。ただし、今回の実用解は XQuartz の入れ替えではなく、GrADS 側で Cairo 表示を避けることです。

gradsx でも黒い場合

gradsx -lp でも画面が黒いままなら、X11 backend への切り替えだけでは足りません。その場合は、まず PNG 書き出しで GrADS 本体の描画が生きているかを確認します。

# GrADS プロンプト内
open your_data.ctl
set gxout shaded
d variable
printim /tmp/grads-check.png png white

PNG が正しく出るなら、当面は printim と Preview を使う運用に切り替えます。Preview は同じファイルを再書き出ししたとき表示が更新されることがあるので、ライブ表示に近い確認用途なら実用になる場合があります。

open /tmp/grads-check.png

それでもライブ表示が必要な場合の次の候補は、XQuartz の別バージョンを試すことです。2.8.6 系はこの記事作成時点では RC なので、研究作業の主環境にいきなり入れるより、戻せる前提で試すのが無難です。

切り分けの要点

  • printim の PNG が正常なら、GrADS の計算・描画命令・地図データは大きくは壊れていない。
  • XQuartz の画面だけ黒いなら、まず grads -d X11 で Cairo 表示 backend を避ける。
  • GADDIR は MacPorts 版に合わせて /opt/local/share/grads を指すようにする。
  • 通常の grads は残し、ライブ表示用だけ gradsx に分ける。
  • gradsx でもだめなら、printim 運用か XQuartz の別バージョン検証に進む。
