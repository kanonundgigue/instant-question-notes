MacPorts経由でGrADSをインストールする方法

作成日: 2026-07-03

更新日: 2026-07-03

カテゴリ: climate-data

元の質問

MacPorts経由でgradsをインストールする方法は？

回答

結論だけなら、MacPortsを入れたうえで sudo port install grads を実行します。MacPortsのportsページでは、grads portはGrADS 2.2.1として公開されており、インストールコマンドも sudo port install grads と案内されています。GrADSは地球科学データの対話的な可視化ツールで、binary、GRIB、NetCDF、HDF-SDSなどの形式を扱うものとして説明されています。[1]

前提を入れる

MacPortsをまだ入れていない場合は、先にAppleのCommand Line Developer Toolsを入れ、macOSのバージョンに合うMacPortsのpkgインストーラを使います。MacPorts公式のQuickstartでも、最初に xcode-select --install を実行し、その後macOS別のMacPortsインストーラを入れる手順になっています。[2]

xcode-select --install

# 新しいターミナルを開き直してから確認
port version

port: command not found になる場合は、新しいターミナルを開き直します。MacPortsのpkgインストーラは通常シェル設定を調整しますが、反映には新しいシェルが必要になることがあります。手動で直す場合は、/opt/local/bin と /opt/local/sbin が PATH の前方に入っているか確認します。[2]

GrADSを入れる

ports treeを最新化してからGrADSをインストールします。MacPortsでは selfupdate がports treeとMacPorts baseの更新に使われ、MacPortsのgradsページでも後日の更新方法として sudo port selfupdate && sudo port upgrade grads が示されています。[3][1]

sudo port selfupdate
sudo port install grads

MacPortsだけでX11サーバまで揃えたい場合は、GrADSの前後どちらかで xorg-server も入れておきます。MacPorts公式のインストール説明では、X11が必要なports向けの選択肢として、MacPortsの xorg-server portが推奨されています。XQuartzを使う構成も選択肢として挙げられています。[2]

sudo port install xorg-server

XQuartzを使う場合は、XQuartz公式サイトのpkgを入れてからXQuartzを起動します。XQuartzはmacOS上で動くX.Org X Window Systemの実装として説明されています。[4]

入ったことを確認する

まずMacPortsの管理下で入ったかを確認します。

port installed grads
port installed active and grads
port contents grads
command -v grads

port installed grads で grads @2.2.1_23 のように出るのに、行末に (active) が付かず、command -v grads も空になる場合は、MacPortsにはインストール済みだが有効化されていない状態です。この場合は次を試します。

sudo port activate grads

それでも /opt/local/bin/grads already exists のように言われる場合は、以前のOpenGrADSなどが残したリンクやファイルが邪魔をしている可能性があります。たとえば /opt/local/bin/grads -> /Applications/OpenGrADS/grads というリンクがあり、リンク先の /Applications/OpenGrADS/grads が存在しない場合、zshからは command not found: grads になります。診断は次のようにします。

ls -l /opt/local/bin/grads
ls -l /Applications/OpenGrADS/grads
port provides /opt/local/bin/grads

port provides /opt/local/bin/grads でMacPorts portの所有物ではないと出る壊れたリンクなら、退避してからMacPorts版を有効化します。

sudo mv /opt/local/bin/grads /opt/local/bin/grads.broken-opengrads-link
sudo port activate grads

グラフィック画面を開く確認は、X11サーバを起動した状態で行います。X11まわりが未設定だと、GrADS本体のインストールは成功していても表示を開く段階で失敗します。

grads

GrADSのプロンプトが出たら、終了は次のようにします。

quit

何が一緒に入るか

MacPortsのPortfileでは、grads は netcdf、hdf4、hdf5、wgrib2、curl、shapelib、xorg-libXmu などに依存し、ビルド設定でもGRIB2、SDF、HDF4、HDF5、NetCDF、X11を有効にする指定が入っています。つまり、MacPorts経由ならデータ形式まわりの依存関係もまとめて管理されます。[5]

GrADSスクリプトはどこに置くか

.gs のGrADSスクリプトは、MacPorts管理下の /opt/local/share/grads には置かず、自分のホームディレクトリかプロジェクトディレクトリに置きます。GrADSのスクリプト実行は run filename で行い、スクリプト名は通常 .gs で終わります。公式ドキュメントでは、run cbar.gs は cbar のように省略できる、と説明されています。[6]

おすすめは次の分け方です。

  • 一時的な解析スクリプト: そのデータや .ctl ファイルのある作業ディレクトリに置く。
  • プロジェクトで使うスクリプト: プロジェクト内の scripts/grads/ や grads/ に置き、Git管理する。
  • cbarn.gs のような共通ユーティリティ: ~/grads/scripts/ のような個人用ディレクトリにまとめる。
mkdir -p ~/grads/scripts
grep -qxF 'export GASCRP="$HOME/grads/scripts"' ~/.zshrc || \
  echo 'export GASCRP="$HOME/grads/scripts"' >> ~/.zshrc

# 反映
source ~/.zshrc

GrADSはまずカレントディレクトリでスクリプトを探し、見つからなければ .gs を付けて再試行し、それでも見つからない場合に環境変数 GASCRP で指定したディレクトリを探します。[7] したがって、普段使いの共通スクリプトだけを GASCRP 側に置き、解析ごとのスクリプトは作業ディレクトリまたはプロジェクト配下に置くのが扱いやすいです。

# カレントディレクトリにある plot.gs を実行
grads
ga-> run plot

# 明示的にパスを指定して実行
ga-> run scripts/grads/plot_map.gs

更新とアンインストール

後日GrADSを更新する場合は、ports treeを更新してから対象portをupgradeします。[1]

sudo port selfupdate
sudo port upgrade grads

削除する場合は、MacPortsの管理対象として次のように外します。

sudo port uninstall grads

つまずきやすい点

  • port が見つからない場合は、新しいターミナルを開き直すか、/opt/local/bin と /opt/local/sbin のPATHを確認する。
  • port installed grads では出るのに grads が見つからない場合は、まず port installed active and grads で有効化状態を見る。
  • /opt/local/bin/grads が古いOpenGrADSへの壊れたリンクなら、そのリンクを退避してから sudo port activate grads を実行する。
  • インストール前には sudo port selfupdate を実行し、古いports treeのまま作業しない。
  • 描画ウィンドウが開かない場合は、GrADSではなくX11サーバ側の問題として、xorg-server またはXQuartzの起動状態を確認する。
  • macOSをメジャーアップグレードした直後は、MacPorts公式のmigration情報も確認する。MacPortsのインストールページでも、macOSのメジャーアップグレード時はmigration情報を見るよう案内されています。[2]
