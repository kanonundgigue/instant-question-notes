pseudo-inverse、backslash、LASSOの違いは？

作成日: 2026-06-25

カテゴリ: statistics

元の質問

pseudo-inverse, backslash, LASSOの違いは？

回答

pseudo-inverse（疑似逆行列）と backslash は、線形方程式や通常の最小二乗問題を数値的に解く方法です。一方、LASSO は L1 罰則を加えて目的関数そのものを変える統計モデルです。

設計行列を X、目的変数を y、係数を beta とすると、Python では概ね次の対応になります。Python には backslash 演算子がないため、通常の最小二乗には NumPy の lstsq を使います。

import numpy as np
from sklearn.linear_model import Lasso

beta_pinv = np.linalg.pinv(X) @ y
beta_lstsq, *_ = np.linalg.lstsq(X, y, rcond=None)

model = Lasso(alpha=0.1, fit_intercept=False)
model.fit(X, y)
beta_lasso = model.coef_

pseudo-inverse（np.linalg.pinv）

通常の逆行列 inv(X) は、X が正方かつ正則な場合にしか定義できません。Moore–Penrose 疑似逆行列 X+ は、長方形行列やランク落ちした行列にも定義できます。

np.linalg.pinv(X) @ y は残差 ||X beta - y||2 を最小にする係数のうち、さらに ||beta||2 が最小のものを返します。つまり、最小二乗解が複数ある場合の選択規則が「係数ベクトルの長さが最小」です。

NumPy の pinv は特異値分解（SVD）を使い、小さすぎる特異値を許容誤差に基づいて 0 とみなします。この切り捨ては数値的不安定性を抑えますが、交差検証で強さを選ぶ LASSO のような統計的正則化とは別物です。

backslash（\、mldivide）

X \ y は「X の逆行列を作って y に掛ける」という意味ではありません。行列の形状や性質を判定し、LU、QR、Cholesky など適切な分解を使って、X beta = y または対応する最小二乗問題を直接解きます。

通常は、逆行列や疑似逆行列そのものが必要でない限り、inv(X) * y や pinv(X) * y より X \ y を優先します。計算量、メモリ、丸め誤差の面で直接解法が有利だからです。

X が過決定かつ列フルランクなら、pinv(X) * y と X \ y は、丸め誤差の範囲で同じ通常最小二乗解になります。違いが重要になるのは、特にランク落ちや劣決定の場合です。その場合、pinv は最小 L2 ノルム解を明示的に選びますが、backslash の解は同じとは限りません。

Python には backslash に対応する演算子はありません。通常は np.linalg.solve(X, y) を正方・フルランクな連立方程式に、np.linalg.lstsq(X, y, rcond=None) を長方形行列を含む最小二乗問題に使います。ただし、NumPy の lstsq は解が複数ある場合に最小 L2 ノルム解を返すため、ランク落ち時の解選択は MATLAB の backslash と一致するとは限りません。

LASSO

LASSO は通常の最小二乗解を別のアルゴリズムで計算しているのではありません。概念的には次の別問題を解きます。

minimize:
    (1 / (2n)) * ||y - X beta||2^2
    + lambda * ||beta||1

lambda が大きいほど係数は強く 0 へ縮み、一部は正確に 0 になります。この性質を変数選択に使えます。その代わり、LASSO の係数には意図的なバイアスが入り、通常最小二乗の残差平方和だけを最小にする解ではありません。

scikit-learn ではこの正則化強度を alpha と呼びます。実務では説明変数のスケールをそろえ、alpha を交差検証などで選びます。切片は通常、L1 罰則の対象外です。

ランク落ちした具体例

2 列が同一で、beta1 + beta2 = 1 なら同じ予測になる状況を考えます。

import numpy as np
from sklearn.linear_model import Lasso

X = np.array([
    [1.0, 1.0],
    [2.0, 2.0],
    [3.0, 3.0],
])
y = np.array([1.0, 2.0, 3.0])

beta_pinv = np.linalg.pinv(X) @ y
beta_lstsq, *_ = np.linalg.lstsq(X, y, rcond=None)

lasso = Lasso(alpha=0.1, fit_intercept=False)
lasso.fit(X, y)
beta_lasso = lasso.coef_

print(beta_pinv)   # [0.5 0.5]
print(beta_lstsq)  # [0.5 0.5]
print(beta_lasso)  # 一方の係数が0になる疎な解
  • np.linalg.pinv(X) @ y は、同じ当てはまりを持つ解の中から最小 L2 ノルムの [0.5, 0.5] を選びます。
  • np.linalg.lstsq も最小 L2 ノルム解を返すため、この例では [0.5, 0.5] になります。これは MATLAB の backslash がランク落ち時に返す解と同一とは限りません。
  • LASSO は alpha に応じて係数を縮小します。同一または強く相関した変数のどちらを残すかは不安定になり得ます。

どれを使うか

  • 通常の線形方程式・最小二乗をPythonで解きたい: 正方・フルランクなら np.linalg.solve、一般の最小二乗なら np.linalg.lstsq。
  • 疑似逆行列そのものが必要: np.linalg.pinv(X)。係数だけが必要なら、通常は行列を明示的に作らない np.linalg.lstsq を使う。
  • 予測の過学習を抑え、係数を縮小し、変数も絞りたい: scikit-learn の Lasso または LassoCV。alpha は検証データや交差検証で選ぶ。
  • 説明変数が強く相関している: LASSO の選択は不安定になり得るため、Ridge や Elastic Net も比較する。

結論

pinv と backslash は、基本的には罰則なしの線形問題をどう数値的に解くかの違いです。LASSO はL1 罰則を追加して、推定したい解の定義を変える方法です。

列フルランクの通常最小二乗なら pinv(X) * y と X \ y はほぼ同じ係数になります。しかしランク落ちすると、どの最小二乗解を選ぶかが違います。LASSO はさらに別の基準で係数を縮めるため、三者を単純な代替コマンドとして扱わないことが重要です。
