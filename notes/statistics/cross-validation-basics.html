機械学習における交差検証とは？

作成日: 2026-06-25

カテゴリ: statistics

元の質問

機械学習における交差検証とは？

回答

交差検証（cross-validation）は、手元のデータを学習用と検証用に繰り返し分け、未知のデータに対するモデルの予測性能を見積もる方法です。略して CV とも呼ばれます。

学習に使ったデータ上の成績だけを見ると、そのデータを過度に覚えたモデルも高く評価されます。交差検証では、各回の学習に使わなかったデータで評価するため、過学習を見つけたり、モデルやハイパーパラメータを比較したりできます。

代表的なk-fold交差検証

最も一般的なのが k-fold交差検証です。データをほぼ同じ大きさの k 個のグループ（fold）に分け、次の処理を k 回行います。

  1. 1つのfoldを検証データにする。
  2. 残りの k - 1 個を使ってモデルを学習する。
  3. 検証データで予測性能を測る。
  4. 検証に使うfoldを入れ替え、すべてのfoldを1回ずつ検証に使う。

最後に、各回の評価値を平均します。たとえば5-fold交差検証では、毎回データの約80%で学習し、約20%で評価します。5回の評価値が 0.81, 0.84, 0.79, 0.83, 0.82 なら、平均スコアは 0.818 です。

何のために使うか

  • 汎化性能の推定: 未知のデータに対する予測性能を見積もる。
  • モデルの比較: 線形モデル、Random Forest、勾配ブースティングなどを同じ分割で比較する。
  • ハイパーパラメータ調整: 正則化の強さ、木の深さ、近傍数などを選ぶ。
  • 安定性の確認: foldによって性能が大きく変わらないかを見る。

平均値だけでなく、fold間のばらつきも確認します。ただし、各foldの評価値は互いに完全には独立でないため、単純な標準偏差をそのまま厳密な信頼区間として扱うことはできません。

ホールドアウト法との違い

方法 分割と評価 特徴
ホールドアウト法 学習用と検証用に1回だけ分ける。 高速だが、偶然選ばれた分割に評価が左右されやすい。
k-fold交差検証 検証部分を入れ替えて k 回学習・評価する。 データを効率よく使えるが、学習計算はおよそ k 倍になる。

交差検証と最終テストデータは別

交差検証で最も成績のよいモデルを選び続けると、選択結果が交差検証のデータに適応していきます。そのため、可能なら最初に最終評価専用のテストデータを分け、モデル選択には使わず最後に1回だけ評価します。

厳密にモデル選択と性能推定を同時に行う場合は、外側の交差検証で性能を評価し、各外側foldの学習データ内でさらに内側の交差検証を行ってハイパーパラメータを選ぶネストした交差検証（nested cross-validation）を使います。

データに合った分割方法を選ぶ

データの性質 主な方法 理由
独立な回帰データ KFold 標準的なk-fold分割を使える。
クラス比が偏った分類 StratifiedKFold 各foldのクラス比を全体と近づける。
同じ人物・地点などの反復測定 GroupKFold 同じグループが学習側と検証側にまたがる漏洩を防ぐ。
時系列データ TimeSeriesSplitなど 未来のデータで学習して過去を予測する不自然な分割を避ける。

無作為なk-fold分割が妥当なのは、基本的に各標本を独立かつ同一の分布から得たとみなせる場合です。空間データ、時系列、患者単位のデータなどには、依存構造に対応した分割が必要です。

前処理もfoldの内側で学習する

標準化、欠損値補完、特徴量選択、主成分分析などを全データに対して先に実行すると、検証データの情報が学習処理に混ざるデータ漏洩が起きます。前処理のパラメータは各foldの学習データだけから求め、検証データにはその結果を適用します。

scikit-learnでは、前処理とモデルを Pipeline にまとめると、この手順を交差検証の各fold内で実行できます。

Pythonの例:

from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import StratifiedKFold, cross_val_score
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler

X, y = load_iris(return_X_y=True)

model = make_pipeline(
    StandardScaler(),
    LogisticRegression(max_iter=1000),
)

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring="accuracy")

print("各fold:", scores)
print("平均:", scores.mean())
print("標準偏差:", scores.std())

fold数はいくつにするか

一般的な出発点は 5-foldまたは10-foldです。k を増やすと1回あたりの学習データは増えますが、学習回数も増えます。全標本を1つずつ検証に使う leave-one-out は計算量が大きく、テスト誤差推定の分散も高くなり得るため、常に優れているわけではありません。

結論

交差検証は、限られたデータを複数の学習・検証分割に使い、未知データへの予測性能を評価する仕組みです。単に cv=5 と指定するだけでなく、最終テストデータをモデル選択から隔離すること、前処理をfold内で行うこと、時系列やグループ構造に合った分割を選ぶことが重要です。
