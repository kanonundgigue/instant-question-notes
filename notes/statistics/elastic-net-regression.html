Elastic Net 回帰とは？

作成日: 2026-06-18

カテゴリ: statistics

元の質問

Elastic Net回帰とは？

回答

Elastic Net 回帰は、線形回帰に L1 正則化 と L2 正則化 を同時に入れる回帰モデルです。L1 正則化を使う Lasso 回帰の「係数を 0 にして変数選択する」性質と、L2 正則化を使う Ridge 回帰の「係数をなめらかに縮めて安定化する」性質を混ぜたものです。

日本語では「エラスティックネット回帰」または「Elastic Net 回帰」と書かれます。

何を最小化しているか

普通の線形回帰は、予測値と実測値のずれが小さくなるように係数を決めます。Elastic Net はそこに、係数が大きくなりすぎることへの罰則を足します。scikit-learn の表記に寄せると、目的関数は概念的に次の形です。

誤差の大きさ
+ alpha * l1_ratio * 係数の絶対値の和
+ alpha * (1 - l1_ratio) * 係数の二乗和 / 2

alpha は正則化全体の強さ、l1_ratio は L1 と L2 の混ぜ方です。l1_ratio = 1 なら Lasso、l1_ratio = 0 なら Ridge に近い形になり、0 < l1_ratio < 1 が Elastic Net らしい中間領域です。

Lasso と Ridge の中間にある

方法 罰則 係数の動き 向いている場面
Ridge 回帰 L2 係数の大きさを全体に縮める。通常、係数は 0 ぴったりにはなりにくい。 説明変数が多く、互いに相関していて、予測を安定させたい。
Lasso 回帰 L1 係数を 0 にできるため、変数選択として使いやすい。 効く変数が少数だと考えられ、モデルを疎にしたい。
Elastic Net 回帰 L1 + L2 変数選択しつつ、相関した変数群をより安定して扱う。 説明変数が多く、かつ相関した変数のグループがある。

なぜ Elastic Net が必要になるのか

Lasso は、不要な変数の係数を 0 にできるので解釈しやすいモデルを作れます。しかし、強く相関した説明変数が複数あると、その中から 1 つだけを選び、残りを捨てるような動きになりやすいです。どれが選ばれるかがデータの揺れに敏感になることもあります。

Elastic Net は L2 成分を加えることで、この不安定さを和らげます。Zou と Hastie の論文では、Elastic Net は強く相関した予測変数が一緒にモデルへ入る、または一緒に外れるような grouping effect を促す方法として説明されています。

パラメータの読み方

実務では、正則化の強さと混合比を分けて考えます。ライブラリによって名前が違うので注意が必要です。

  • 正則化の強さ: scikit-learn では alpha、glmnet では主に lambda。大きいほど係数が強く縮みます。
  • L1 と L2 の混合比: scikit-learn では l1_ratio、glmnet では alpha。1 に近いほど Lasso、0 に近いほど Ridge です。
  • 標準化: 罰則は係数の大きさにかかるため、説明変数のスケールが違うと比較が歪みます。通常は学習前に説明変数を標準化します。

Python での最小例

Python で scikit-learn を使う場合は、説明変数を標準化してから ElasticNetCV で alpha と l1_ratio を交差検証で選ぶのが基本形です。

from sklearn.linear_model import ElasticNetCV
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler

model = make_pipeline(
    StandardScaler(),
    ElasticNetCV(l1_ratio=[0.1, 0.5, 0.9, 1.0], cv=5, random_state=0),
)

model.fit(X_train, y_train)
pred = model.predict(X_test)

l1_ratio=[1.0] だけを試すなら実質的に Lasso 寄りです。相関した説明変数があり、Lasso の選択が不安定に見える場合は、0.1、0.5、0.9 のような中間値も候補に入れると比較しやすくなります。

使うときの注意

  • 係数が 0 になったからといって、その変数が因果的に不要だと決まるわけではありません。
  • 相関した変数群では、どの変数が残るかだけでなく、予測性能と係数パターンの安定性を見ます。
  • 目的が推論なら、正則化後の係数をそのまま通常の回帰係数のように検定・信頼区間解釈するのは危険です。
  • 目的が予測なら、訓練データの当てはまりではなく、交差検証や検証データで比較します。
  • カテゴリ変数、外れ値、非線形性、交互作用の扱いは、Elastic Net だけでは解決しません。前処理と特徴量設計も必要です。

結論

Elastic Net 回帰は、Lasso の疎な変数選択と Ridge の安定化を組み合わせた正則化回帰です。説明変数が多く、しかも互いに相関しているときに特に有用です。短く言えば、「Lasso だけだと不安定になりやすい高次元・相関ありの回帰で、変数選択と予測安定性を両立させたいときの選択肢」です。
