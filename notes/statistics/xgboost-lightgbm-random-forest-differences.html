XGBoost・LightGBM・Random Forestの違いは？

作成日: 2026-06-24

カテゴリ: statistics

元の質問

XGBoost・LightGBM・Random Forestの違いは？

回答

最も大きな違いは、Random Forest は独立に作った多数の木を平均するバギング、XGBoost と LightGBM は前までの予測誤差を次の木で順番に補正する勾配ブースティングだという点です。

そのため、Random Forest は比較的調整しやすく堅牢な基準モデルになりやすい一方、XGBoost と LightGBM は適切に調整すると表形式データで高い予測性能を狙いやすい反面、学習率、木の複雑さ、反復回数などの調整がより重要になります。

3手法の要点

手法 学習の仕方 実務上の性格
Random Forest データと特徴量をランダムに変えながら多数の木を独立に作り、分類は多数決、回帰は平均を取る。 調整が比較的容易で、ノイズや過学習に対して堅牢。まず置く基準モデルに向く。
XGBoost 損失関数の勾配と2階微分を使い、前の予測を改善する木を逐次追加する。目的関数に木の複雑さへの正則化を含む。 精度、正則化、機能、計算効率のバランスがよい。堅実な勾配ブースティングの候補。
LightGBM 勾配ブースティングをヒストグラム化と leaf-wise 成長で高速化する。GOSS や EFB も提案された。 大規模・高次元データで高速かつ省メモリになりやすい。小規模データでは過学習制御が特に重要。

Random Forestだけ仕組みの系統が違う

Random Forest では、各決定木をブートストラップ標本で学習し、各分割でも候補となる特徴量をランダムに絞ります。互いに完全には同じでない木を多数作り、その予測を平均することで、単一の深い決定木が持つ高い分散を下げます。

各木は基本的に独立して作れるため並列化しやすく、木を増やしてもブースティングのように前の木の結果を待つ必要はありません。また、学習に使われなかった out-of-bag データを利用して、別の検証データを用意せずに汎化誤差を概算できます。

XGBoostとLightGBMは誤差を順番に補正する

XGBoost と LightGBM は、どちらも Gradient Boosting Decision Tree（GBDT）の実装です。1本目の木で残った誤差を2本目、そこでも残った誤差を3本目というように、木を逐次追加して損失を下げます。

Random Forest の主な狙いが「木どうしの平均で分散を下げる」ことであるのに対し、勾配ブースティングの主な狙いは「弱い木を順次足してバイアスを下げる」ことです。ただし、実際には行・列のサンプリング、正則化、木の深さなども使うため、バイアスと分散の両方を制御します。

XGBoostとLightGBMの違い

両者の差は「ブースティングかどうか」ではなく、主に木の作り方と高速化の設計にあります。

  • XGBoost: 勾配だけでなく損失の2階微分も使い、葉の重みや木の複雑さに正則化を入れた目的関数を最適化します。現在の公式実装では高速なヒストグラム法が標準で、深さ優先の depthwise と損失減少優先の lossguide を選べます。
  • LightGBM: 連続値をビンにまとめるヒストグラム法を中心に、最も損失を減らせる葉を次に分割する leaf-wise 成長を使います。同じ葉数なら損失を下げやすい一方、一部の枝だけが深くなり、小さいデータでは過学習しやすくなります。
  • LightGBM固有の高速化: 原論文では、大きな勾配を持つ標本を重視する GOSS と、同時に非ゼロになりにくい疎な特徴量をまとめる EFB が提案されています。

「XGBoost は level-wise、LightGBM は leaf-wise」とだけ説明されることがありますが、これは現在では単純化しすぎです。XGBoost も設定によって loss-guided な成長が可能で、両者ともヒストグラム法を使えます。既定値と実装上の設計思想が違う、と理解する方が正確です。

どれを選ぶか

状況 最初の候補 理由
まず堅牢な基準を作りたい Random Forest 前処理とハイパーパラメータ調整への依存が比較的小さく、単純な決定木より安定しやすい。
精度を詰めたい一般的な表形式データ XGBoost 正則化と学習制御が充実し、速度と精度のバランスを取りやすい。
行数・特徴量数が多く、学習速度やメモリが課題 LightGBM ヒストグラム法と leaf-wise 成長により、大規模データで効率がよいことが多い。
データが少なく、過学習が心配 Random Forest または制約を強めた XGBoost LightGBM の leaf-wise 成長は、葉数・深さ・葉の最小データ数を適切に制限しないと深い枝を作りやすい。

比較するときに固定すべき条件

どの手法が最も高精度かはデータ次第です。名前だけで決めず、同じデータ分割と評価指標で比較します。

  • 交差検証の分割、乱数シード、評価指標をそろえる。
  • クラス不均衡がある場合は、accuracy だけでなく ROC-AUC、PR-AUC、F1 など目的に合う指標を使う。
  • XGBoost と LightGBM は、学習率と木の本数をセットで調整し、検証データによる early stopping を使う。
  • Random Forest は、木の本数だけでなく max_features、葉の最小サンプル数、木の深さを確認する。
  • カテゴリ変数や欠損値のネイティブ対応は、手法名ではなく使用するライブラリとバージョンで確認する。

結論

Random Forest は独立な木を平均する、調整しやすく堅牢なバギング手法です。XGBoost は正則化と2階情報を組み込んだ汎用性の高い勾配ブースティング、LightGBM は leaf-wise 成長などで大規模データでの効率を重視した勾配ブースティングです。

実務では、Random Forest を基準モデルにし、XGBoost と LightGBM を同じ交差検証条件で比較するのが確実です。中小規模で堅実に詰めるなら XGBoost、大規模で速度とメモリが問題なら LightGBM が有力ですが、最終判断は検証データ上の性能、計算時間、安定性で行います。
