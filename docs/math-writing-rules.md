# 数式の記述・レイアウト規約

このファイルは、返答・記事・ドキュメントに数式を含める場合だけ参照します。数式を使わない作業では読みません。

## 適用する場面

次のいずれかを新規作成または修正するときに、この規約を適用します。

- 数式、導出、確率分布、行列・ベクトル、添字付き変数を含む文章
- 数学記号の組版が必要な HTML 記事
- 数式を含む Markdown またはチャット回答

日付、単位付きの単純な数値、コマンド名、ファイル名、プログラム上の識別子だけを書く場合は対象外です。

## 基本ルール

- 数式のソースは LaTeX で記述する
- 数式を `<code>` で囲まない
- Unicode 記号や `<var>`、`<sub>`、`<sup>` の組み合わせで数式を組み立てない
- プログラム上の識別子や実行コードは従来どおり `<code>` またはコードブロックを使い、数式と区別する
- 変数は本文で先に定義し、式だけを置いて説明を省略しない

## 区切り記号

HTML 記事では次を使います。

- インライン数式: `\(...\)`
- 独立数式: `\[...\]`

Markdown とチャット回答では、表示環境が対応している場合に次を使います。

- インライン数式: `$...$`
- 独立数式: `$$...$$`

HTML 記事では `$...$` や `$$...$$` を使いません。

## MathJax

数式を含む HTML 記事だけで MathJax を読み込みます。数式のない記事には追加しません。

```html
<script>
  window.MathJax = {
    tex: {
      inlineMath: [["\\(", "\\)"]],
      displayMath: [["\\[", "\\]"]]
    }
  };
</script>
<script src="https://cdn.jsdelivr.net/npm/mathjax@3.2.2/es5/tex-chtml.js" defer></script>
```

記事固有の数式 CSS は書かず、`styles/main.css` の共通設定を使います。長い独立数式は次の共通設定により、ページ全体ではなく数式の内部だけを横スクロールできるようにします。

```css
.article mjx-container[display="true"] {
  box-sizing: border-box;
  max-width: 100%;
  overflow-x: auto;
  overflow-y: hidden;
  padding: 0.2rem 0;
}
```

## 読みやすい配置

- 短い変数や短い関係式はインライン数式にする
- 分数、総和、期待値、行列、複数段の等式は独立数式にする
- 1本の式に変形を詰め込まず、1行につき1つの論理的な変形を置く
- 複数行では `aligned` を使い、等号や不等号の位置をそろえる
- 長い式は、等号、加減算、積の境界など意味の切れ目で改行する
- フォントを小さくして押し込まない
- 長い数式を表の狭いセルへ入れない。表の外に独立数式として置く
- 式の前後に十分な説明を置き、記号の意味と式が表す操作を文章で示す

HTML 内で `aligned` の位置合わせに使う `&` は、HTML として妥当になるよう `&amp;` と書きます。

```html
<p>
  \[
    \begin{aligned}
      \ell(\theta)
      &amp;= \log p(x \mid \theta) \\
      &amp;= \log \sum_z p(x, z \mid \theta)
    \end{aligned}
  \]
</p>
```

## 記号の書き方

- 対数は `\log`、総和は `\sum`、期待値は `\mathbb{E}` を使う
- 条件付きを表す縦線は `\mid` を使う
- 正規分布は `\mathcal{N}` のように、意味に合う書体を使う
- 最大化演算子などは `\operatorname` または `\operatorname*` を使う
- 転置は文脈に応じて `\mathsf{T}` など、記事内で表記を統一する
- 添字と上付きは役割が分かるようにし、同じ記号へ複数の意味を割り当てない
- δ（デルタ）は同位体比専用とし、他の物理量には使わない

## 引用と数式

- 外部資料に基づく式は、式を説明する文の末尾に通常の上付き引用番号を置く
- 引用番号を MathJax の区切り記号の内側へ入れない
- 式を転載するだけでなく、記号の定義とその式を使う条件を説明する

## このリポジトリ固有の注意

`npm run check` は `{{` と `}}` をテンプレートプレースホルダとして検出します。LaTeX の入れ子の波括弧で `}}` が連続する場合は、外側の閉じ波括弧の前に LaTeX 上無視される空白を入れます。

```latex
\mu_k^{\mathrm{new} }
```

表示を変えずに検査を通すための措置なので、変更後は必ず MathJax の描画結果も確認します。
