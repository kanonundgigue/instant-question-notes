# AGENTS.md

このリポジトリは、疑問や調べたことを静的 HTML の記事として蓄積する「instant-question-notes」です。エージェントはこのファイルを最初に読み、詳細が必要なときだけ [docs/agent-operations.md](docs/agent-operations.md) を参照します。

## 最優先ルール

- 返答、記事、README などのドキュメントは日本語で書く
- ユーザーが疑問や調べたいことだけを送った場合、明示がなければ公開記事作成依頼として扱う
- 「説明だけ」「相談だけ」「記事にしないで」「差分を見たいだけ」と言われたら記事化しない
- 機密情報、個人情報、未公開の認証情報を記事・コミット・返答に含めない
- ビルドツールや静的サイトジェネレータを導入しない
- 作業ブランチや PR は、ユーザーが明示的に求めた場合だけ使う
- Python を使う場合は `.python-version` に従い、パッケージ管理は `uv`（`uv run`, `uv sync`, `uv pip install`）を使う。bare `pip` / `python -m venv` は使わない
- Docstring・コメントは日本語で書き、Python には型ヒントを付ける
- δ（デルタ）は同位体比専用にし、他の物理量には使わない

## 記事作成の標準手順

1. 質問内容からカテゴリ名とスラッグを決める
2. 調査が必要な内容は一次情報を確認する
3. `notes/_template/article.html` を元に `notes/<category>/<slug>.html` を作る
4. `index.html` の新着、カテゴリ導線、カテゴリ別セクションを更新する
5. 記事カードは更新日がある記事は更新日、ない記事は作成日を基準に日付降順で並べる
6. 検証結果と差分要約を提示して止まる。コミットは `commitして` と言われたときだけ行う

既存記事を実質的に更新した場合は、記事ページと `index.html` のカードに更新日を追加または更新します。表記ゆれや HTML 整形だけなら更新日は不要です。

## 記事の必須要素

- `<h1>`: 質問の主旨を表すタイトル
- メタ情報: 作成日、必要なら更新日、カテゴリ、元の質問文
- 本文: `<h2>` / `<h3>` で構造化する
- 参照: 調査した場合は記事末尾に一次情報の URL を列挙する
- ナビゲーション: トップページへの戻りリンクを置く

## HTML / CSS

- 記事固有の `<style>` は原則使わず、共通 CSS は `styles/main.css` に置く
- トップページからの CSS パスは `styles/main.css`
- 記事ページからの CSS パスは `../../styles/main.css`
- 画像を追加する場合は `images/` 配下に置く
- テーブルは原則 2-3 列に抑え、`<table class="mobile-table">` と各 `<td data-label="...">` を使う
- 横スクロール前提の幅指定、長い非改行テキスト、列数の多い表を避ける

## 保護記事

- 記事は既定で公開する。保護するのは、ユーザーがコミット前に「この記事は保護して」と明示した場合だけ
- 保護記事は `notes/private/` に置き、入口一覧 `notes/private/index.html` にカードを追加する
- 暗号化は `tools/encrypt.html` を localhost 経由で開いて行う
- 共通パスワードは `PROTECTED_NOTES.local.md` から読み取り、返答やコミットに書かない
- 平文下書きは絶対にコミットしない。必要なら `_drafts/` または `*.draft.html` に置く
- 保護記事は復号確認と平文漏れ確認を行う

## Git / 公開

- ローカル運用では、指示されない限りコミットしない
- `pushして` / `公開して` と明示された場合だけ `main` に直接 push する
- push 前に `git status` で対象差分を確認し、無関係な差分を含めない
- `git pull`, `git merge`, `git rebase`, `git checkout <branch>`, `git reset`, `git restore`, `git switch` などローカル変更が上書きされ得るコマンドの前には、必ず `git status` を確認し、未コミット変更があれば先に `git add` + `git commit -m "WIP: auto-save before <操作名>"` を行う
- コミットメッセージは、新規記事なら `add: <カテゴリ>/<スラッグ> <記事タイトル>`、既存記事修正なら `fix: <対象> <内容>`、構成変更なら `chore: <内容>`

## 削除禁止

`rm`, `rmdir`, `unlink`, `trash`, `git clean` は実行しません。削除が必要な場合は `cleanup.sh` に理由コメント付きで削除コマンドを追記し、ユーザーに確認を依頼します。`cleanup.sh` は自分では実行しません。

## 公開前チェック

作業完了前、特に commit / push 前に、可能な範囲で以下を確認します。実行できなかった項目があれば理由を報告します。

- `git status --short --ignored` で未追跡・無視ファイル・無関係差分を確認する
- `git diff --check` で空白エラーを確認する
- `npm run check` で HTML、リンク、保護記事、モバイル表の基本検査を行う
- テンプレートの `{{ }}` プレースホルダが残っていないことを確認する
- `PROTECTED_NOTES.local.md`、`*.draft.html`、`_drafts/`、`node_modules/`、`outputs/` をコミット対象にしない
- 保護記事の平文下書き、ローカル秘密ファイル、生成作業ディレクトリが公開対象に含まれないことを確認する
- 新規記事へのリンクが `index.html` から辿れることを確認する
- 保護記事を触った場合は `note-payload` と復号 UI が壊れていないことを確認する
